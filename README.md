# EPL Como Fantasy - 26/27

Config-driven **live auction-draft** tool for the Como fantasy league. Rerun it each season or sport by swapping `league.config.json`.

> **v1 = the auction, in person, Aug 2 2026.** Season scoring stays on the official FPL Draft site - this app never scores the season.

**Status (2026-07-15):** the v1 feature beta is deployed from [`b9d1c5b`](https://github.com/Kolam-Studios/epl-como-fantasy-26-27/commit/b9d1c5b) on [Vercel](https://epl-como-fantasy-26-27-cgtd.vercel.app) with Neon and the real roster override. Core auction flow, corrections, trades, sealed reveals, recap, read-only trade log, phone fixes and expanded standard player stats are implemented. The integrated runner contains 16 isolated suites. Feature presence is not final production acceptance: remaining Aug 2 gates are production reset/freeze, the post-reset audit and the formal two-device, sustained-load and physical/fallback rehearsal.

Requirements live in `docs/PRD.md`; verification methods and latest evidence live in `docs/TEST-PLAN.md`; [issue #9](https://github.com/Kolam-Studios/epl-como-fantasy-26-27/issues/9) is the live delivery tracker; [issue #23](https://github.com/Kolam-Studios/epl-como-fantasy-26-27/issues/23) records deployment execution. ## What it is

- **The war room.** No logins. One open web page serves the TV, laptops, and phones: current lot, every budget, every live max bid, every squad, the remaining pool, the sale log. Bidding is verbal in the room; a neutral auctioneer records outcomes in a token-gated console - the app is the system of record, not the auctioneer.
- **Two-phase auction.** Phase 1 offers every player once (price order, shuffled within tier; no-bids wait). Phase 2 is nomination in fixed rotation until every squad is full.
- **The rules engine.** Tiered opening bids, a max-bid reserve rule so nobody can strand themselves short of a full squad, exclusive ownership enforced in the database, undo plus edit/void of any sale with an audit trail, and in-auction trades where salaries travel with players.
- **The theatre.** A sealed fair-price valuation per player (generated by Claude on draft morning, calibrated to the league economy) is hidden from everyone until the hammer, then revealed beside the price paid with an OVERPAY / FAIR / STEAL verdict. Morning news briefs appear on the block. `/recap` implements the three awards, each manager's leftover war chest, final position-grouped squads, per-device FPL Draft entry checklists and a direct ledger link, with an idempotent archive.
- **Future:** v2 in-season rebid rounds; v3 historical archive import.

Everything above is configuration, not code: budget, squad shape, tier bands and opening bids, manager count. The committed defaults ($3,000 budget, 8 managers, 15-player squads, four tiers opening $50/$25/$10/$5) are just this league's settings in `league.config.json`.

## Stack

- **Next.js** (App Router, TypeScript) - runs on Vercel natively. One app serves the board, squads, ledger, recap, console and API. `output: standalone` supports future container packaging, but this repo does not currently include a Dockerfile.
- **Postgres** (Neon / Supabase free tier, or self-hosted) - single `DATABASE_URL`.
- **Live updates by polling** `/api/state` (~2s). No WebSocket infra; ports anywhere unchanged.
- **FPL public API** for the player pool; **Claude API** for the draft-morning briefs + valuations (optional - the auction runs without them).

## Deploy targets

- **Production:** [Vercel](https://epl-como-fantasy-26-27-cgtd.vercel.app) against Neon Postgres.
- **Verified laptop fallback:** `npm run build` then `npm start` against the same Postgres database. No Dockerfile is present. See `docs/DEPLOYMENT.md`, `docs/PORTING.md` and `docs/HANDOFF.md`.

## Quick start

```bash
cp .env.example .env.local          # set DATABASE_URL + COMMISSIONER_TOKEN
npm install
npm run db:setup                    # apply db/schema.sql, seed managers
npm run ingest                      # pull FPL player pool -> players table
npm run dev                         # http://localhost:3000
```
