# EPL Como Fantasy - 26/27

Self-hosted **live auction-draft** tool for the Como fantasy league. Config-driven framework: reruns each season/sport by swapping `league.config.json`.

> **v1 = the auction, in person, Aug 2 2026.** Season scoring stays on the official FPL Draft site - this app never scores the season.

**Status (2026-07-08):** decisions locked, build underway. The plan of record is [issue #9](https://github.com/miloli-git/epl-como-fantasy-26-27/issues/9); the approved requirements are `docs/PRD.md` + `docs/DESIGN.md` + `docs/DATA-MODEL.md`. Agents: read `CLAUDE.md` first.

## What it is

- **The war room.** No logins. One open web page serves the TV, laptops, and phones: current lot, every budget, every live max bid, every squad, the remaining pool, the sale log. Bidding is verbal in the room; a neutral auctioneer records outcomes in a token-gated console - the app is the system of record, not the auctioneer.
- **Two-phase auction.** Phase 1 offers every player once (price order, shuffled within tier; no-bids wait). Phase 2 is nomination in fixed rotation until every squad is full.
- **The rules engine.** Tiered opening bids, a max-bid reserve rule so nobody can strand themselves short of a full squad, exclusive ownership enforced in the database, undo plus edit/void of any sale with an audit trail, and in-auction trades where salaries travel with players.
- **The theatre.** A sealed fair-price valuation per player (generated by Claude on draft morning, calibrated to the league economy) is hidden from everyone until the hammer, then revealed beside the price paid with an OVERPAY / FAIR / STEAL verdict. Morning news briefs on the block. A recap/awards page after the night.
- **Future:** v2 in-season rebid rounds; v3 historical archive import.

Everything above is configuration, not code: budget, squad shape, tier bands and opening bids, manager count. The committed defaults ($3,000 budget, 8 managers, 15-player squads, four tiers opening $50/$25/$10/$5) are just this league's settings in `league.config.json`.

## Stack

- **Next.js** (App Router, TypeScript) - runs on Vercel natively and self-hosts in Docker identically. One app serves the board, squads, ledger, recap, console, and API.
- **Postgres** (Neon / Supabase free tier, or self-hosted) - single `DATABASE_URL`.
- **Live updates by polling** `/api/state` (~2s). No WebSocket infra; ports anywhere unchanged.
- **FPL public API** for the player pool; **Claude API** for the draft-morning briefs + valuations (optional - the auction runs without them).

## Deploy targets

- **Reference:** self-hosted via Docker, pointed at Postgres (container or Neon).
- **Port:** `vercel deploy` against the same schema. No WebSocket server and no local SQLite file, so the port is a deploy-target swap - see `docs/PORTING.md` and `docs/HANDOFF.md`.

## Quick start

```bash
cp .env.example .env.local          # set DATABASE_URL + COMMISSIONER_TOKEN
npm install
npm run db:setup                    # apply db/schema.sql, seed managers
npm run ingest                      # pull FPL player pool -> players table
npm run dev                         # http://localhost:3000
```
