# Decisions - RESOLVED

> Status: **CLOSED (2026-07-07).** Every open item below was decided by the league owners after a full design walkthrough; all three owners have approved. The requirements of record are `docs/PRD.md`, `docs/DESIGN.md`, and `docs/DATA-MODEL.md` - if anything here drifts from those, they win. This file is kept as the decision log.

## The decision table

| # | Topic | Decision |
|---|---|---|
| 1 | Budget | **$3,000 per manager** (config default) |
| 2 | Managers | **8** (count always derives from the config array; real roster only in gitignored local config) |
| 3 | Squad | **15 = 2 GK / 5 DEF / 5 MID / 3 FWD**; exclusive ownership enforced as a DB constraint |
| 4 | Tiers / opening bids | **Four tiers by FPL price band, opening $50 / $25 / $10 / $5** (T1 >= 12.0, T2 >= 9.0, T3 >= 7.0, T4 rest). Bands, opens, and increments all config-editable; the old "$1 floor, no increments" is superseded |
| 5 | Lot order | FPL price descending, **shuffled within tier**. No nomination in phase 1. Console sees ~5 lots ahead; the room learns the next name only when the current lot closes |
| 6 | Auction structure | **Two phases.** Phase 1: every player offered once; no bid = marked NO BID, stays available, no ad-hoc requeue. Phase 2 (auctioneer ends phase 1 manually): **nomination in fixed rotation**, skipping full squads, until all squads are 15/15 |
| 7 | Max bid | `remaining budget - (minimum opening bid x other open slots)`, min opening bid = lowest tier's open ($5 default, config-driven). Computed live for all 8, displayed everywhere, enforced at sale entry |
| 8 | Bidding model | **Commissioner-entry (decided - no longer open).** Voice bidding in the room, no timers, no in-app bidding; a neutral auctioneer records outcomes in the console |
| 9 | Access model | **War room: no logins, no private data, no watchlists.** One open page for TV, laptops, phones; shows only publicly derivable numbers |
| 10 | TV control | The console selects the board view (block / reveal / squads / ledger / paused); all screens poll shared state (~2s) |
| 11 | Corrections | Undo last sale; **edit or void ANY past sale** from the ledger; every change writes a visible audit trail |
| 12 | In-auction trades | **v1 must-have.** Entered by the auctioneer during a pause: players and/or cash. Salaries travel with players; cash settles differences. Guardrails: no negative budgets, quotas respected, <= 15 players. Board announces trades |
| 13 | AI features | **Sealed valuations** per player, generated by Claude on draft morning, calibrated to the league economy; hidden from everyone until the hammer, then revealed with an OVERPAY / FAIR / STEAL verdict (sealed server-side). **Morning news briefs** for top tiers, on demand for the rest. Sale grading is relative to the live market, thresholds configurable |
| 14 | Prior-season owner line | The board shows the on-block player's prior Como owner + price; hides gracefully if the data is absent |
| 15 | Post-auction | Recap/awards page, per-manager FPL Draft entry checklist, permanent archive |
| 16 | Visual design | Locked - see `docs/VISUAL-DESIGN.md` |
| 17 | Scoring | **Never in the app.** The official FPL Draft site runs the season |
| 18 | Roadmap | v1 = auction night (Aug 2 2026). v2 = in-season rebid rounds (parameters TBD with the group). v3 = historical archive import |

Also resolved along the way: repo is public (`github.com/Kolam-Studios/epl-como-fantasy-26-27`, MIT); pool source is FPL `bootstrap-static`; formation/pitch view is out of v1 (squads are slot counts during the auction); real names never enter the repo (placeholder config + gitignored local roster).

Hosting is resolved: Vercel is the cloud primary, with a verified `npm run build` plus `npm start` laptop fallback. The physical fallback drills remain pending.
